프롤로그 | 시간을 다르게 써야 인생이 바뀐다

제1장 시간 관리는 인생 관리다 
  01. 나에게 맞는 시간 관리법 찾기 
    - 시간 관리법의 두 가지 흐름
      ; 효율을 추구하는 시간 관리법 -> "해야 할 일"을 얼마나 "효율적" 으로 하는가
      ; 만족감을 추구하는 시간 관리법 -> "하고 싶은 일" 을 얼마나 "만족스럽게" 하는가 

    - 시간 관리에서 가장 먼저 알아야 할 것 
      ;단계를 나누고 시간관리를 진행 - 단계를 나누지 않으면 "무엇을 해야 할지" 고민하게 되면서 시간과 에너지를 낭비하게 됨
      ; 계획-> 실행 -> 점검의 순으로 진행 
      ; 계획 - 효율성과 만족감의 비중을 잘 조율해야함 
                효율만을 생각해서는 안 되고 만족감의 비중을 조절하여 동기부여의 면도 고려해야 함
      ; 실행 - 실행은 집중력 문제 
                 실행 단계의 준비가 중요함 : 어떤 마음가짐으로 실행에 임할지, 실행에 앞서 집중에 방해가 된느것을 어떻게 제거 통제할지 결정해야 함
      ; 점검 - 내가 스스로 평가자가 되어 내 계획과 실행을 점검 : 계획과 실행의 기록이 필요함
                 문제점을 인식 -> 수정 -> 반영하는 세부단계로 시간관리에 대한 점검을 진행 

제2장 완전한 자유는 완벽한 계획으로부터 나온다
  01. 하고 싶은 일과 해야 하는 일 정리하기 
    - 시간 관리에 대해 생각하는 순서
     ; 시간 관리에 대한 이미지 - 시간 관리를 이미지 화 하여 방을 꾸미듯이 설계 
     ; 시간 관리가 어려운 이유 - 시간을 이미지 화 한것에 배치할때 어려움을 격는다 (사람의 기억력)
     ; 단계를 나누는 습관 - 단계를 나누어 한번에 한 가지 일만 처리하는 것이 중요, 세부적으로 단계를 나누어 '내가 현재 하는 일이 무엇인지' 명확하게 
                                    만드는것이 중요 

    - 떠오르는 일을 모두 적어보기
     ; 떠오르는 일을 "하고싶은일", "해야할 일"로 나누어 적어보자, (모든일을 일단 적는다) 
     ; "만다라트 계획표" : 3x3 칸에 할일을 적고 세부 항목으로 늘려간다. (한번 작성해볼 것)
 
    - 미래 목표에서 현재로 거꾸로 거슬러 오라 
    ; 목표를 세우고 계획할때 세부단계로 설계하며 시간 및 할당량을 계획에 명시하는 것이 중요함 
      ex) 책을 많이 읽겠어 (X), 책을 하루에 12페이지는 읽겠어 (O)
    ; 계획중 장기계획과 단기계획을 나누어 관리할 필요가 있다.

    - 다짐과 계획을 구별하라
    ; 계획을 수치화 하여야 한다 -> 수치화를 통하여 달성 및 미달성에 대하여 점검할수 있음 
    ; OKR(Objective and Key Results) 목적 + 핵심성과 -> 목적은 추상적이지만, 핵심성과는 수치화가 가능함 
    ; SMART 원칙 : 
                      1. Specific : 구체적이고     
                      2. Measurable : 수치화가 가능하여 
                      3. Achievable : 그 계획의 달성 여부를 쉽게 알 수 있어야 하며
                      4. Realistic : 비현실적이지 않고 
                      5. Time-bound : 기한이 존재해야 한다

    - 일의 체계를 만드는 방법
     ; 장시간 동안 해야 하는 일이거나 제법 규모가 있는 일 같은 경우 그 일들 사이에 체계를 만들어두면 
       진행상황을 점검하고 잘못된 점을 수정해나갈 때 훨씬 편리하게 일 을 할 수 있다.
     ; 목적과 수단의 구별 : 목적을 두고 , 그것을 달성하기 위한 세부 수단 명시 
     ; 5W2H 질문법 :  
                          WHY? -> WHAT? -> Who, When, Where, How, How Much
                         1. Why : 가용성 산출물 작성 
                         2. What : 위험평가 대상 선정하기 
                                      위험평가 계획서, 결과서 작성하기 
                                      연간 백업계획 대상선정 및 계획서 작성 요청하기 
                                      이중화테스트 대상선정 요청하기 
                                      이중화테스트 대상관련 회의 
                                      이중화테스트 연간 계획서 작성 
                                      가용성 연간 계획서 작성 
                         3. who? : 나
                         4. when? : 2023. 3. 31 :최종 산출물 제출 
                                       2023. 2. 27. :연간 백업계획 대상선정 및 계획서 작성 요청하기 
                                                         3년간 이중화 히스토리 정리, 2022년 주요장애 정리
                                       2023. 2. 28. : 이중화테스트 대상선정 요청하기
                                       2023. 3. 22. : 이중화테스트 대상선정 회의
                                       2023. 3. 23. :  이중화테스트 연간 계획서 작성
                                       2023. 3. 27. : 가용성 연간 계획서 작성
                         5. where? : 회사 사무실
                         6. How? : 위험평가 대상 산출 -> 전체CI 목록에 서비스 카탈로그 정보로 점수 산출 후 최종 0.6점이상 대상 선정 
                                       위험평가 계획서, 결과서 작성 -> 2022년 계획서 참고  
                                       연간 백업계획 대상선정 및 계획서 작성 요청 -> 정영선B에게 요청 
                                       이중화테스트 대상선정 요청하기 -> 3년간 이중화 히스토리 정리, 2022년 주요장애 정리, 위험평가 결과서 첨부하여 
                                                                                        각 파트PL 전달 후 대상 선정 요청 
                                       이중화테스트 대상선정 회의 -> 전달 받은 목록 중 사법 2, 등기 2 대상선정(3년간 이중화 히스토리 및, 2022년 주요장애 참고)
                                       이중화테스트 연간 계획서 작성 -> 2022년 계획서 참고, 대상선정 회의 도출 내용 정리 후 반영 
                                       가용성 연간 계획서 작성 -> 연간 백업계획 대상, 연간 이중화 계획을 바탕으로 작성, 혹시 서비스 카탈로그 변경시 내용 변경 필요 
                         7. How Much? : 비용 무, 주요 일정마다 업무시간 2시간이상 할당
      ; 설득하는 일에서 고려할 점(자소서, 제품 홍보등) : 
                         1. 상대방이 원하는 것이 무엇인지 확실하게 파악한다. 
                         2. 상대방이 원하는 것에 부합하는 내가 가진 강점이 무엇인지 파악한다.
                         3. 상대가 볼때 나의 단점이라고 생각될 수 있는 점에 대하여 생각해 본 후 그 에 대한 반론 및 대비  
                         4. 반론 및 대비에는 이유나 근거를 제시하여야 함(계획에 반론 및 대비에 대한 이유나 근거 데이터베이스 구축을 포함) ???? 

    - 계획은 반드시 최선과 최후 두 가지로 세울 것
     ; 계획은 반드시 플랜 A(최선) 과 플랜 B(최후) 두 가지를 세운다. 

  02. 우선순위 정하기 
    - 시간 관리의 고정 '아이젠하워 원칙'
     ; 일을 "중요성"과 "긴급성"을 기준으로 4등분하여 관리 
       (현재 내가 집중하여 처리해야 할 일이 무엇이지 판단, 중요한 일은 퀄리티를 고려하는 원칙)
        ① 중요하며 긴급한일(RED) -> 즉시 직접처리 
        ② 중요하지만 긴급하지 않은일(BLUE) -> 미루어 두었다가 직접 처리 
        ③ 긴급하지만 중요하지 않은 일(Green) -> 위임, or 일 처리가 더딘 시간때(오침 후 등) 
        ④ 중요하지도 긴급하지도 않은 일 -> 버리기, 생략 
     ; 추가되야 할 기준 - "얼마나 손쉽게 일을 처리할여 일의 개수를 줄일 수 있는지"의 문제. 일의 "간이성" 기준 추가 
      => 긴급하고 중요한 일이 여러개면 쉬운거 부터 처리하여 에너지를 쏟아야 하는 일을 마지막으로 처리한다. 
          (마지막 한개 남았다 라는 마음가지을 가지고 에너지를 쏟을 일읍 처리할 수 있다) 
     ; 중요한 일인지 판단하는 법 - "내 인생에서 중요한 가치가 무엇인지 생각해보는 과정이 없이는 어떠한 일이 내 인생에서 더 중요한지 판단하기 어렵다"
       ※※ 중요 (필수진행) ※※ 
      => 내 인생에서 중요한 가치 7가지 나열하기
      => 중요성을 판단하기 위한 질문 답하기 
       ① 당신의 삶에서 가장 중요한 일은 무엇인가요?
       ② 가족과의 여가와 업무 성과 중에 무엇이 더 중요한가요? 
       ③ 당신은 이기적인 사람인가요, 이타적인 사람인가요?
       ④ 개인과 공동체 중 무엇을 더 중요하게 생각하나요?
       ⑤ 지금부터 10년 뒤에 달성하고 싶은 목표가 있나요?
       ⑥ 죽기 전에 반드시 하고 싶은 일이 있나요?
        
    - 중요성과 긴급성에 따른 우선순위 매기기
       ① 중요하며 긴급한일(RED) -> 내가 정한 중요한 가치에 비추어 판단할 때, 최선의 결과를 내야하는 중요한 일 또는 
                                            당장처리하지 않으면 이익을 얻지 못하거나 불이익을 받을 우려가 있는 일 
       ② 중요하지만 긴급하지 않은일(BLUE) -> 결과물이 중요 내가 최고의 컨디션일 때 직접처리 해야 하지만 지금 당장 하지 
                                            않아도 불이익이 없는 일 ex) 자기개발 
       ③ 긴급하지만 중요하지 않은 일(Green) -> 1, 2순위보다 덜 중요하나 처리하지 않으면 이익을 얻지 못하거나 불이익이 큰 경우 
       ④ 중요하지도 긴급하지도 않은 일 -> 하고 싶은 일 ex) 게임, 영화보기  

    - 일의 효능성을 판단하여 쳐내기
      ; 계획을 세우고 나서 내가 맞는 일인지, 시간 낭비인지 확인 필요 -> 5W2H방법 활용
      ; 일이 주는 이익과 그로 인하여 포기하게 될 불이익에 집중하여 두 가지를 비교 
    - 우선순위에 따라 계획 재정리하기

    - 요약

  03. 일정 속에 배치하기 
    - 종류에 따라 계획표 준비하기 
     ; 장기, 단기 계획에 따라 계획표를 달리 준비한다. 
     ; 종이 + 인터넷(스마트폰)을 활용한 계획표 세우기 
       추천 툴 - > 노션, 에버노트, 콜라비, 잔디, 슬랙, 줌, 마인드노트, 구글 태스크, 구글 킵, 토큐먼트, 미로, MS Teams, MS Project, MS SharePoint, 드롭박스

    - 장기계획표에 확정된 일정부터 배치하기 
     ; 장기계획 또는 확정될 일정을 구글 캘린더에 기입한다.

    - 중요하고 긴급한 일의 배치
     ; ※파레토의 법칙에 따른 배치 - 최상의 효율을 보이는 시간에 배치한다. -> 시간 + 집중
         ※파레도의 법칙 : '전체 업무 시간 중 20%의 시간대에 전체 80%의 성과가 나온다'
     ; 다음날 아침 전날 일정을 피드백 하는 시간을 갖자 ★★★★★

    - 중요하지만 긴급하지 않은 일의 배치
     ; 적립식 시간 사용 방법에 따른 배치 - 하루 중 일정한 시간들을 모아두는 습관 (방해받지 않을 수 있는 시간)
     ; 오전 일찍 기상에 따른 시간, 퇴근 후 시간, 업무중 방해 받지 않을 시간 등에 배치 
     ; 출퇴근을 대중교통을 이용하고 그시간을 이용할 수 있을지 확인 -> 현재는 유투브로 활용중(운전시)

    - 중요하지 않지만 긴급한 일의 배치
     ; 묶어서 처리하는 것이 중요 1순위 처리 후 남는시간 (2순위는 먼저 적립식 시간 사용방법에 따라 할당된 시간에 진행)
     ; 일을 맡기는 경우 - 조직내에서 방향성이나 관리 검증을 주로 신경쓰며, 일을 팀원에게 분배하여 효율성을 올린다. 
     ; 일을 직접처리하는 경우 - 메일 회신시간을 정한다. 회의시간을 정한다.
    
   - 중요하지도 긴급하지도 않은 일의 배치
     ; 1~3순위 일을 배치하고 남는시간에 진행 
 
    - 나만의 마감일을 설정할 것
     ; 일을 모두 배치하였으면 데드라인을 정하여 실행 - 데드라인 이펙트: 데드라인이 되면 집중력이 올라감 
     ; 하기 싫은 일은 일부러 자진해서 맡아라 - 도전하라 !! ★★★
     ; 데드라인 이펙트 - 나와의 약속. 하기싫은일 맡기 - 다른 사람과의 약속 
     ; 구글 타이머 활용

    - 일의 연속성을 고려할 것
    ; 일은 워밍업 시간이 필요함으로 되도록 계획을 세울때 연속적으로 세워야 효율적이다. 
    ; 쿨링 다운을 하여서 효율을 높여야 한다 - 지속적인 일은 효율을 떨어뜨림 
    ; 컴프리헨서블
   - 멀티태스킹
    ; 첫째 : 단기집중력을 필요로 하는일을 하면서 자투리 시간에 장기집중력(상대적으로 집중이 적은 일)을 같이 하는 방법 - 고민해 보자
    ; 둘째 : 하나의 일을 하면서 동시에 여러 가지 목표를 달성할 수 있는 경우 
    ; 컴프리헨서블 인풋 or 다이렉트니스 러닝 - 하고싶은일을 다이렉트로 하면서 세부적인 사항을 찾아나가는 방법 
      ex) 노마트 코더 코딩을 하면서 세부 항목을 배워나간다, 영어회화를 하며 필요한 문법부분을 찾아본다.

    - 요약

     ; 구글 타이머 등을 이용하여 시간을 직관적으로 보고 관리하라 

    - 일단 시작하라
     ; 일단은 앉는 습관과 일단을 시작하는 습관을 만들어라 
     ; 성취동기가 없으면 의욕이 없을수 있다. 쉬운일로 시작하여 점차 강도를 올린다.
     ; 끊임없이 자신의 결단(목표)에 책임질 수 있도록 노력하라 

    - 나를 끌어주는 말과 감사로 하루를 시작하라
     ; 오늘의 다짐과 함께 시작하는게 도움이 될 수도 있다. (긍적적인 사고와 동기부여)

    - 방해물을 미리 제거하여 최적의 환경을 만들라
     ; 내 목표를 방해하는 요인을 찾는것이 중요하다 - 주된적은 술 ★★★★★
     ; 무의미한 '습관화된 시간 낭비짓'이 어떤 원리로 작동하는지 살펴봐야 한며, 이러한 습관이 동작하지 않게 결단해야 한다.
     ; 작은 것이라고 생각한 한번에 예외가 열번 스무번이 되고 결국 계획을 망친다.  ★★★★★

    - 실행의 성패는 사전 시뮬레이션에 달려 있다.
     ; 계획을 세우고 실행전 시뮬레이션 과정을 거친다 - 머릿속으로 떠올려봐도 별다른 어려움 없이 계획 전체가 떠오르고 해야할
      일이나 우선순위가 빠짐없이 생각날 정도 
     ; 이러한 과정을 통하여 실행도중 다음으로 해야 할 일을 생각하는 상황이 발생하지 않도록 한다. 
    - 요약

  02. 실행단계에서 기억해야 할 점들 
    - 실행단계에서 효율을 높이는 방법
     ; 세부계획을 짜고, 내가 할 일을 쪼개보는 것 필요 
     ; 빠르게 원하는 자료, 정보를 찾을 수 있게 정리 하는 것이 필요
     ; 반복적으로 진행 되는일에 대해서는 루틴화 필요 

    - 일의 템포와 타이밍
     ; 일의 템포는 빠르게 가져가는 것이 좋다. 다만 퀄리티가 떨어지는 것은 방지 
     ; 템포를 빠르게 가져가지만 타이밍은 적절하게 가져가서 보고나 제출하는 것이 좋음 
       (너무 빠른 타이밍은 상대로 하여금 신뢰감 정성도가 없다고 판단될 수 있음)
     ; 간트 차트 활용하여 시간관리
 
    - 업무위임을 하는 경우 효율을 높이는 법
     ① 그 일이 위임이 가능한 일인지 분별한다. (내가 해야 효율이 있는 것은 직접 처리)
     ② 상대방의 능력과 의욕을 고려하여 위임의 강도 조절 
        ; 능력 의욕 모두 뛰어나면 수단 선택, 실행 등을 모두 맡기고 최종 결과만 받는다
        ; 능력은 좋지만 의욕이 떨어지는 사람(일 잘해 일이 몰리느 사람)은 실행 부분은 맡기되 정신적, 물직적 격려나 응원필요
        ; 의욕은 넘치나 능력이 떨어지는 경우 가이드라인을 잡아주며 가르치는 느낌으로 일  위임 
        ; 능력 의욕 모두 떨어지면 맡기지 않는것이 좋으나 필요시 매우 구체적으로 지시하고 그 일에 대한 결과를 매 세부 업무마다 받는다.
     ③ 업무위임을 하는 목적을 상대방에게 공유해야 하며, 상대방이 해야 할 일이 명확해야 한다.
         지시는 추상적이어서는 안 되고 구체적이고 상세해야 한다. 
     ④ 업무위임의 기한을 정한다. (팀원의 시간을 고려해서 결정 - 필수 ★★★)

    - 업무위임 받을 때 주의사항 
     ① 자신이 할 수 없는 일은 거절하는 것이 훨씬 바람직하다. (이성적인 계산으로 맡지 못할 경우 예의를 갖춰 거절)
     ② 일을 맡을 경우 실행하기 전에 다시한번 일의 목적, 일의 범위, 내용 등을 확인해야 한다. 

    ※ 새로운 일을 맡았을 때 빠르게 익히는 법 (차세대 ACI에 적용) 
      ; PAGE 165 참조 
    - 보고시 주의사항 
      ; 상대방에게 미리 그에 소요될 예상시간을 알려준다.
      ; 주제와 결론, 핵심논거, 향후조치 순으로 핵심정보만 요약하여 전달한다.
      ; 예상되는 질문에 대한 답까지 미리 준비 
  
    - 피드백 결재시 주의사항 
      ; 긍정적인 피드백을 할 시 결재는 명확해야 하고 가급적 신속해야 한다.
      ; 결제가 느려지는 이유 
       ① 상대가 보고한 일 중 어느 부분이 위임취지대로 잘되었고 잘못되었는지 가릴 능력이 없음
       ② 능력은 있지만 잘못된 부분에 대한 대안이나 보완 지시를 못하는 경우
       ③ 보완이 별 필요 없어 바로 결재할 수 있음에도 우유부단함에 결정하지 못하는 경우
       ④ 보고자체가 엉망인 경우  
     ; 결제를 할 경우 결과에 대하여 수용하고 믿어라 
     ; 결과에 대한 부분은 지시한 사람 결제자가 전적으로 책임져야 한다. 

     - 회의시 주의사항 
     ① 아이디어를 모으는 회의 - 1차적으로 폐기하며, 아이디어를 모아야하는 의제에 대하여 
          화이트 보드나 미로보드에 각자가 그때 그때 떠오르는 아이디어를 붙이고 시간이 지나면서 취합되게 만든다 
     ② 2개 이상의 안 중에 결정하는 회의 - 미리 서로가 지지하는 안을 공유한 후 각자 반론과 재반론을 정리한 후 회의 진행
     ③ 진행상황이나 일의 결과 등의 정보 공유 회의 - 얼굴을 마주보고 하는 회의보다, 메신저나 게시판 등을 통해 공유 
           실제 상황공유 할 수 있는 방법을 도입하는게 유리함 
           (EX> 주간회의 협업툴을 통한 보고 공유 내용에 대한 숙지여부에 따라 보상 또는 패널티 적용) 
     ④ 회의록 작성 필수 - 클로바노트 사용 

    - 실행 도중의 버퍼링을 해결하는 방법
     ① 현재 할일이 무엇인지 명확하게 모를때 - 세부계획을 재점검 해본다
     ② 하는 일의 방향성에 대한 의구심 - 방향성에 대한 재점검 
     ③ 동력을 상실한 경우 - 휴식으로 에너지를 보충 

    - 기준을 가지고 거절하라
     ; 거절하지 못하는 것은 마음이 약하거나 상대방을 신경쓰는게 아니다. 
       실상은 내 행동에 명확한 기준을 가지고 있지 않거나 계산력이 매우 부족하다는 의미다. 
       (기준을 명확히 하는 것이 중요)
     ; 거절을 못하는 것은 나를 배려하지 않는 행동, 즉 나와의 약속을 깨는 행동이다.
     ; 기준을 명확이 하는 법
       ① 부탁, 요구가 있을 때 수락 거절 하기전 멈추어 생각하는 습관을 만든다. 
       ② 현재 내가 할 일의 양과 시간을 생각해본다.
       ③ 내 일에 지장이 없다면,  승낙시 나와 상대방에게 발생할 이익, 거절시 나와 상대방에게 발생할 불이익을 생각해본다.
       ④ 비교 결과 거절을 해야 하는 상황이면 "샌드위치 기법"에 따라 거절한다. 
        ※ 샌드위치 기법 (PAGE 174)
         - 기분 나쁘지 않은 말 -> 정말로 하고 싶은 말(거절) -> 다시 기분이 나쁘지 않은 말로 마무리 
         ex) 커피 마시러 갈래 ? -> 커피 마시는 거 너무 좋지 근데 내가 지금 너무 급하게 할일이 있어서 나중에 마시러 갈 때 한 번 더 예기해줄래?
 
    - 돌발 상황이 발생했을 경우의 마음가짐 - 
     ; 돌발 상황이 일으키는 부정적인 감정을 철저히 배제해야 한다. - 최대한 현 상황을 긍정적으로 보려고 노력한다.

    - 완벽주의는 실행의 최대의 적
     ; 판단 주체를 헷갈렸을 때 - 완벽한 결과의 판단은 내가 아닌 상대방이 되어야 한다. 
     ; 판단 기준이 헷갈렸을 때 - 상대방의 기준을 파악하여고 노력해야 한다. 기준 미달, 기준 이상이 되면 의미가 없다.
        
    - 요약

제4장 점검과 재충전도 시간 관리의 일부다
  01. 점검하기
    - 시간 관리의 끝은 점검이다.
	- 문제점 파악이 전체 결과를 좌우한다.
	- 면밀하게 따져보되 과감하게 수정하라
	- 실제로 바뀌지 않는다면 점검은 아무런 의미가 없다.
	- 기록과 점검의 시기
	- 요약
  02. 재충전하기 
    - 재충선이 시간 관리의 일부인 이유
	- 육체적인 재충전과 정신적인 재충전
	- 일을 시작하기 전 단계에서의 재충전 
	- 일하는 과정에서의 재충전
	- 일을 끝낸 후의 재충전
	- 감정을 컨트롤 하는 법 
	- 스트레스를 없애는 방법
	- 요약
